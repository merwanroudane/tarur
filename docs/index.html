The Problem

Why TARUR?

Researchers applying nonlinear unit root tests face a recurring frustration: existing implementations in R (e.g., NonlinearTSA) do not provide critical values, forcing manual lookups in the original papers for every test. TARUR eliminates this entirely.

📊

Embedded Critical Values

Every test uses its own CVs from the original paper (asymptotic + finite-sample). Automatic linear interpolation for intermediate sample sizes.

✅

Automatic Decisions

Reject / fail-to-reject at 1%, 5%, and 10% — with a plain-English interpretation sentence — printed automatically for every test.

🔬

Bug Fixes Over R Code

Corrects known errors in NonlinearTSA: Kruse's modified Wald τ, Hu-Chen's 3-parameter statistic, Sollis (2009) auxiliary regression.

⚡

One-Line Full Battery

tarur.run_all_tests(y) runs every available test at once and produces a unified comparison table — ideal for papers.

📄

LaTeX Export

Call batch.to_latex() to get a ready-to-paste LaTeX table for your journal submission — no manual formatting.

📈

Publication Visualizations

Built-in diagnostic dashboards, test statistic comparison charts, and decision heatmaps via batch.plot().

Installation

Getting Started

TARUR requires only the standard scientific Python stack. No R, no compiled extensions.

🚀 Quick Example

import numpy as np
import tarur

# Your time series (log-levels for financial data)
y = np.log(your_price_series)

# ── Single test ────────────────────────────────────────
result = tarur.kss_test(y, case='demeaned', max_lags=12)
print(result)
# → Prints statistic, CVs, decision, interpretation

# ── Full battery (17 tests at once) ────────────────────
batch = tarur.run_all_tests(y, case='demeaned', max_lags=12)
print(batch)           # Formatted table
batch.plot()           # Dashboard
print(batch.to_latex()) # LaTeX for your paper
✓ Requires: numpy ≥ 1.21, scipy ≥ 1.7, pandas ≥ 1.3, matplotlib ≥ 3.5 — all standard, no additional installation needed if you use Anaconda.

Recommended Practice

Researcher Workflow

Follow this four-step protocol for a rigorous empirical application.

import numpy as np
import tarur

# Step 1 — Load data in log-levels
y = np.log(your_price_series)

# Step 2 — Test for linearity first (Teräsvirta 1994)
lin = tarur.terasvirta_test(np.diff(y), d=1)
print(f"p-value: {lin.extra['p_linearity']:.4f}")
print(f"Suggested: {lin.extra['suggested_model']}")  # ESTAR or LSTAR

# Step 3 — Full nonlinear unit root battery
batch = tarur.run_all_tests(y, case='demeaned', max_lags=12)

# Step 4 — Export for paper
df = batch.summary()                           # pandas DataFrame
print(batch.to_latex())                         # Ready-to-paste LaTeX table
batch.plot()                                    # Publication figure
tarur.plot_series_analysis(y, title='My Series') # Diagnostic dashboard
